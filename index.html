adadelta

今天读了一篇paper, AdaDelta, an adaptive learning rate method,是Zeiler在google实习时候所做.
这篇paper试图解决的问题在于,当使用sgd的时候我们并不能及时地调整learning rate(以下简称lr),使得lr
相对于理想中的最佳lr即不偏大,也不偏下,因为偏大的话,就有可能使得结果发散,偏小的话会使得收敛变得缓慢.
因此作者试图找到一种方法使得lr能够自适应地更新.

我们知道sgd使用的是一介梯度信息,而牛顿法则还需要二阶信息,也即需要计算hessian矩阵的逆矩阵.这个逆矩阵
不好求,因此出现了一些列求他近似的方法,本文也是一种这样的方法.

该方法的优点是

  1. 不用手动调整lr
  2. 对超参数不敏感
  3. 参数的每个维度都可以有自己的特定lr
  4. 相较于sgd,计算量并没有增加多少
  5. 对与大梯度,噪声以及不同的模型选择具有鲁棒的特性

具体来说,相对于sgd
delta(x) = -lr grad
adadelta 采用的是
delta(x) = -(RMS(delta(x)))/ RMS(grad) grad
其中RMS(y) = sqrt(ACC(y) ^ 2 + epsilon)
而ACC是相应量的的累加和, epsilon是为了保证做分母的时候整体不为0

整体的计算过程为, 维护两个累加和,关于grad一个,关于最终更新的变量的一个
(1) 首先计算梯度grad, 然后计算 ACC(grad^2) = row ACC(grad ^ 2) + (1-row) grad ^ 2
(2) 计算两个RMS,并且计算delta(X)
(3) 计算ACC(deltaX ^ 2) = row ACC(deltaX^2) (1-row) * deltaX ^2
(3) 依据deltaX,更新X

需要注意的是第二步中两个RMS的计算,由于本次的deltaX此时并没有计算出来,使用的是此步以前的deltaX.以下是caffe中关于这个的实现.

boost_tuple

缘起１

经常使用python童鞋对这样的语句对这样的语句不会感到陌生，如
a, b, c = sys.argv[1:]
意思是将程序的三个输入的参数分别赋给a, b, c.
(a, b) = fun()
是将函数fun的两个返回值分别赋给a, b
(a, b) = (b, a)
是交换a, b 的值

python这里能完成这么多类似的功能的原因是python的数据结构tuple

缘起2

在c++中，有时候我们写的函数的返回值有多个的时候，我们可以有好几种处理方法，
比如如果返回值的类型全部相同，那我们大可以将他们全push到一个vector里面
或者如下使用引用或者指针的方法　将要返回的值通过参数的形式传递进去
void fun(int a, int& p)
甚至于我们可以自己实现一个类或者使用已有的类(如pair)来保存所有的返回值.
可是平心而论，这里所说的处理方式多有不变．我们多么希望能有类似于python tuple那样的
数据结构啊

然后,boost带来了tuple
为了使用tuple,需要包含头文件 boost\/tuple\/tuple.hpp,
tuple是pair的泛化,最多可以将10中类型不尽相同的数据压制在一起
template<class T0, ..., class T9>
class tuple
{
tuple();
tuple(T0 t0, ...);
tuple(const tupe<...>& );
tuple& operator=(const tuple&);
tuple& operator=(cosnt std::pair&);
template<int N> T get();
};

tuple也支持嵌套定义如
typedef tuple<int, int> TP1;
typedef tuple<int, TP1> TP2;

除了通过各种构造函数生成tuple, tuple类库提供类似于make_pair()的函数make_tuple(),如
make_tuple(string(), vector<int>()); //tuple<string, vector<int> >
make_tuple生产的数据默认是无引用,如果要使用引用,则要用到ref库的ref()和cref()

可以通过t.get()来获取tuple t第id个分量,然而这里的id必须是编译器就要知道的值,如下面代码不能work
for(int id=0; id!=3; ++id)
{
std::cout << t.get<id>();
}
类似与此,boost还提供了一个get<>()函数访问tuple的分量,如
get<1>(t)

如果tuple的元素能够执行比较操作的话,tuple也可以进行比较.跟pair的比较方法类似,如果要使用比较
需要包含头文件boost\/tuple\/tuple_comparation.hpp

tuple也支持输出到流,类似于比较操作,tuple输出到流时候只不过是将tuple的元素逐个用operator<< 或者operator>>.
要使用输出功能,需要包含头文件boost\/tuple\/tuple_io.hpp,如
tuple<int, double, string> t1(1, 2.2, "hello");
cout << t1 << endl;
cin >> t1;

这样屏幕会输出(1 2 hello),并且等待新的输入
这里需要注意的是使用cmd输入流的时候,需要包含前导符,后导符,以及分隔符,并且后导符跟最后一个tuple分量之间
要用空格隔开.前导符倒不需要
可以使用方法 set_open(char) set_close(char) set_delimiter(char) 来设置这三种符号

最后,boost tuple提供了一个函数tie(),使得boost 的tuple表现离python的tuple更近一步,如
int i; double d; char c;
tie(i, d, c) = fun();
tie应用与std::pair也是可以的
tie(i, s) = make_pair(100, "string");
如果并不想使用tuple的某个分量,那么可以使用tuples::ignore来占位,
tie(tuples::ignore, s) = make_pair(100, "hellp");

那么如何像在python中那样优雅地交换两个tuple分量的值呢?
tie(a, b) = make_tuple(b, a); //work
tie(a, b) = tie(b, a); //actually not work

#用途
首先gflag是一个command line parser，是用来解析命令行输入参数的。
例如输入命令行 ./< your app > -year 2016 -month 4

#如何解析？
首先在全局初始化的时候可以这样定义一个变量
DEFINE_bool(flag, true, “help flag”);
上述语句定义了一个bool变量flag，默认值为true，变量帮助说明为 help flag的语句。在程序中，即可通过访问FLAGS_flag
这个变量来访问命令行穿进来的flag的值。是不是很方便？
除了bool值，gflag还支持其他5种数值类型，如int32 int64 uint64 string double。那么是否支持更复杂的类型呢？回答是no，google
给出的解释我觉得很赞–他们更愿意将对数据的处理放在程序中，而这时候gflag就只负责解析命令行的参数即可。

近来在使用caffe实现gru unit时候需要书写梯度反向传播的部分代码，而总是弄混caffe中关于gemm的部分代码
因此需要将blas彻底弄清楚，才不会浪费更多的时间，是以记之。

blas以及他的各种衍生包的安装舍去不表。

blas 是 Basic Linear Algebra Subprograms的简写，主要进行的是矩阵运算（向量也算特殊的矩阵）。
因此按照维度的增加，blas将运算分为三个level

  • 向量与向量的运算
  • 矩阵和向量的运算
  • 矩阵和矩阵的运算

（由于太多，现在只了解用到部分，其余部分随用随添加）

  1. level1 向量向量运算
    cblas_?asum ？可取s, d, sc, dz 作用是Sum of vector magnitudes （参数）
  2. level2 矩阵向量运算

  3. level3 矩阵矩阵运算
    cblas_?gemm ？ 可取s, d, c, z 用作一般矩阵的乘法

    alpha beta都是scalar A,B,C是矩阵 A（m×k） B（kxn） C（mxn）
    如果transa=CblasNoTrans, op(A) = A， transa=CblasTrans, then op(A) = AT，transa=CblasConjTrans, then op(A) = AH.
    m 指定的是op（A）以及C的行， n指定的是op(B)和C的列， k是op（A）的列和op（B）
    lda 之类指定的是相应向量的leading demension 比如rowmajor下，transa=CblasNoTrans，lda>=max(1, k)
