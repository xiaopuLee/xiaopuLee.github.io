2016-04-30

今天友人来访，应邀写一个段程序处理数据，其中设计到字符串的操作（split find）
然后发觉自己对这一块儿蛮生疏，查阅cplusplus,总结如下。

string是std的一个类.typedef basic_string string;

  1. 构造函数，copy构造函数，assign

    string() ; //empty string
    string(const string& str); //copy
    　　> string(const string& str, size_t pos, size_t len = npos);
    string(const char s)
    string(const char     s, size_t n)
    string(size_t n, char c);
    template string(InputIterator first, InputIterator last) //[first, last)

    string& operator=(const string& str);
    string& operator=(const char* s);
    string& operator=(char c)

    string& assign(const string& str);
    string& assign(const string& str, size_t subpos, size_t sublen);
    string& assign(const char s)
    string& assign(const char     s, size_t n)
    string& assign(size_t n, char c)
    template string assign&(InputIterator first, InputIterator last);

  2. size相关的
    　　> size_t size() const;

    size_t length() const;
    size_t capacity() const;
    size_t max_size() const;

    void reserve(size_t size = 0);

    bool empty();

    void clear();

    size() 和　length()都返回的是字符串的大小，但更推荐用前者。capacity() 返回的是为当前字符串申请的空间的大小，这个
    大小是以byte来记，并且这个大小不小于字符串的长度。max_size()返回的是字符串能申请到的最大长度。因为字符串可以动态改变大小
    所以，为了更加有效率，当可以预先知道字符串的大致长度范围的时候，最好使用reserve函数申请合适的空间长度，防止字符的不停
    拷贝。
    empty() 返回的是字符串是否为空，当然你可以使用size() <= 0来判断，但是还是推荐使用这个函数。

  3. element access

    char& operator;
    const char& operator;
    char& at(size_t pos);
    const char& at(size_t pos) const;

    operator[] pos 小于length,不报异常。pos等于length时候，const 版本不报异常。其他情况均报异常。
    　　 at 只要不在length范围内，就报异常。

  4. modifers

    string& operator+=(const string& str);
    string& operator+=(const char& c);
    string& operator+=(const char* s)

    string& append(const string& str);
    string& append(const string& str, size_t npos, size_t sublen);
    string& append(const char s);
    　 > string& append(const char     s, size_t n);
    string& append(size_t n, char c);
    template string& append(InputIterator first, InputIterator last);

    string& replace (size_t pos, size_t len, const string& str);
    string& replace (iterator i1, iterator i2, const string& str);
    string& replace (size_t pos, size_t len, const string& str, size_t subpos, size_t sublen);
    　 > string& replace (size_t pos, size_t len, const char s);
    string& replace (iterator i1, iterator i2, const char     s);
    string& replace (size_t pos, size_t len, const char s, size_t n);
    string& replace (iterator i1, iterator i2, const char     s, size_t n);
    string& replace (size_t pos, size_t len, size_t n, char c);
    string& replace (iterator i1, iterator i2, size_t n, char c);
    template string& replace (iterator i1, iterator i2, InputIterator first, InputIterator last);

    string& erase (size_t pos = 0, size_t len = npos);
    iterator erase (iterator p);
    iterator erase (iterator first, iterator last);

    void swap (string& str);

    string& insert (size_t pos, const string& str);
    string& insert (size_t pos, const string& str, size_t subpos, size_t sublen);
    string& insert (size_t pos, const char s);
    string& insert (size_t pos, const char     s, size_t n);
    string& insert (size_t pos, size_t n, char c);
    void insert (iterator p, size_t n, char c);
    iterator insert (iterator p, char c);
    template void insert (iterator p, InputIterator first, InputIterator last);

  5. find相关的
    　 > size_t find (const string& str, size_t pos = 0) const;

    size_t find (const char s, size_t pos = 0) const;
    size_t find (const char     s, size_t pos, size_t n) const;
    size_t find (char c, size_t pos = 0) const;
