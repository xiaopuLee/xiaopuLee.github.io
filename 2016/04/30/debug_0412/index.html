使用gdb进行调试

#缘起
要在ubuntu下工作，又不太想把程序都放在window上写再来回转着调试，所以就想联系在linux系统里使用gdb进行调试

#相信使用

  1. 编译时加入debug信息, 使用参数-g即可。
    g++ -g main.cpp -0 main

  2. 启动测试
    可以使用gdb main，使用-tui 参数可以分屏显示代码，比参数l好用的要死
    也可以使用 gdb 打开调试窗口后，使用file app来打开app进行调试

    set args 来设定程序运行的参数
    也可以gdb之后，使用run <参数列表>
    show args显示参数

  3. 各种参数
    l list 列出当前行前后个5行代码 list n1 n2 列出n1 n2 之间的代码
    c continue 继续运行
    r run
    b break 设置断点 后面可以接行号 以及函数名字；也可以条件断点break 46 if testsize==100；break file：N 设置file的N行为断点
    p 打印后面变量的名字， print &var 打印变量的地址；如果a是一个数组，10个元素，如果要显示则： print *a@10
    n next 单步运行一次， next N，执行下N步

    s 跳入函数内部
    finish 执行完当前运行的函数，返回调用他的函数
    j jump 5 跳转到第五行执行完毕之后，如果之后没有断点，则继续执行
    wathch i！=10
    info breakpoints 简写info break 会显示当前所有的断点，断点号断点位置。
    delete N 删除N号断点； delete 删除所有断点
    clear N 清除行N以上所有断点。
    whatis var 显示变量var的类型
