今儿来说说谷歌单元测试库gtest
（备不住估计还得继续说说gmock，虽然我几分钟前才知道gmock)

这里有一系列博客说gtest，并且说的很好。（http://www.cnblogs.com/coderzh/archive/2009/04/06/1426755.html）

1 Basic concept
使用gtest首先得学会写断言（Assertions)。
断言总是成对出现，会有EXPECT*版，也会有ASSERT*版，前者会产生nonfatal错误并且继续运行，而后者会产生fatal error并且中断
程序。下举一小例。
　　`
ASSERT_EQ(alpha.size(), beta.size()) << “the sizes of alpha and beta are not equal”；

for(int id=0; id!=n; ++id)
{
EXPECT_EQ(a[i], b[i]) << “array a, b not equal at: “ << id;
　　}
`

以下是所有的断言

ASSERT_TRUE EXPECT_TRUE ASSERT_FALSE EXPECT_FALSE(condidtion)
　　> ASSERT_EQ EXPECT_EQ (a, b)
ASSERT_NE EXPECT_NE
ASSERT_GT EXPECT_GT
ASSERT_LT EXPECT_LT
ASSERT_GE EXPECT_GE
ASSERT_LE EXPECT_LE

　 > ASSERT_STREQ EXPECT_STREQ

ASSERT_STRCASEEQ EXPECT_STRCASEEQ
ASSERT_STRNE EXPECT_STRNE
ASSERT_STRCASEEQ EXPECT_STRCASENE
上述a, b 必须是可比较的，可以是build-in type，也可以是自己创建的类型（当然提供可比较的方法）
ASSERT_EQ可以用来对指针进行操作，例如当对比连个c string时候，他比较的是两个指针是否指向的是内存中的同一位置，而不知比较他们的值，
因此要比较两个c string的值，最好使用ASSERT_STREQ，而比较两个string，使用ASSERT_EQ即可。
2016/02以前，如×_EQ(expected, actual)描述的，两个参数一般是前边是期望值，后面是真实值。现在已经完全等同对待了

注意NULL 和空字符串是不同的。

  1. Simple Tests
    使用宏TEST()来创建测试，如下main.cpp
    　　`
    #include
    #include

    int fib(int a, int b)
    {
    if(a < b)

    return -1;

    if(a == b)

    return 0;

    if(a > b)

    return 1;

    }

    TEST(fib, handleAll)
    {
    EXPECT_EQ(fib(1, 2), -1);
    EXPECT_EQ(fib(1, 1), 0);
    EXPECT_EQ(fib(2, 1), 1);
    }

    int main(int argc, char** argv)
    {
    testing::InitGoogleTest(&argc, argv);
    return RUN_ALL_TESTS();
    }
    `
    使用g++ -Wall main.cpp -lgtest -lpthread 即可生成可执行程序。（gtest 和pthread 的顺序一定不能错）
    　　TEST(test_case_name, test_name) {
    　　　… test body …
    　　　　}
    　　中test_case_name, test_name 均是为了输出测试结果时候使用，不要加上引号。

3.Test Fixtures
当我测试想测试自己写的类的时候，如果使用２中的步骤，那么我需要重复地创建类的对象并且做相应的初始化，这很繁琐。所幸gtest可让我们重复使用同一个
数据初始化。
　要完成这些，你需要首先实现一个类，这个类需要继承　::testing::Test, 然后在这个类中声明你需要的东西，如果有必要给这个类加上构造函数
或者在void SetUp()里面准备数据。注意这里的U是大写。以及相应的析构函数或者void TearDown()函数。

　然后就可以仿照２中书写测试样例了，唯一区别的地方是这次使用的TEST_F(test_case_name, test_name)这个宏，并且这里的test_case_name一般得是
你的类名，
　如，我们这边有一个类实现的是队列
template <typename E> // E is the element type. class Queue { public: Queue(); void Enqueue(const E& element); E* Dequeue(); // Returns NULL if the queue is empty. size_t size() const; ... };
然后定义FixtureTest
　class QueueTest : public ::test::Test { protected: virtual void SetUp() { q0_.Enqueue(0); q1_.Enqueue(1); q2_.Enqueue(2); } //virtual void TearDown() //{} Queue<int> q0_; Queue<int> q1_; Queue<int> q2_; }
　　然后就可以创建测试case了
　　`
TEST_F(QueueTest, IsEmptyInitially) {
EXPECTEQ(0, q0.size());
}

TESTF(QueueTest, DequeueWorks) {
int* n = q0.Dequeue();
EXPECT_EQ(NULL, n);

n = q1_.Dequeue();
ASSERT_TRUE(n != NULL);
EXPECT_EQ(1, *n);
EXPECT_EQ(0, q1_.size());
delete n;

n = q2_.Dequeue();
ASSERT_TRUE(n != NULL);
EXPECT_EQ(2, *n);
EXPECT_EQ(1, q2_.size());
delete n;

}
`
这里说明一下，首先gtest会创建一个QueueTest对象Q1来做第一个测试，然后TearDown,销毁这个对象。然后
再次创建一个QueueTest对象Q2来做第二个测试，然后继续TearDown，销毁．上述两个测试例子的顺序可以随意．

需要注意的是
1) #include
2) ::testing::InitGoogleTest(&argc, argv);
3) return RUN_ALLTESTS(); //main函数必须这样返回
　　4) g++ main.cpp -lgtest -lpthread　//两个库的顺序一定不能搞错
5) 如果在函数中使用这个，ASSERT*() 只能用在返回值为void的函数中．

想吐个槽，为啥感觉gtest gflag中argc argv　有时候带&，有时候不带呢，很容易记混啊记混啊，虽然我知道去看相应的源码
就可以知道答案，但是懒得去仔细研究了，这些工具能先好好使用再说了．

今天来说说protobuf，也是谷歌的开源项目，用起来非常好用。 并且所定义的数据结构可以在多种语言之间无差别共享
只要使用的是同一个proto文件，官方支持的语言包括c++ python java c# go，其他语言也可以找到相应的开源版本。

它的作用跟xml差不多，但比xml体积小，速度快。缺点也有，如必须配合相应的proto文件才能看懂里面字段的含义。

起一个文本文件foo.proto，写入下面的内容。定义一个message
Message SearchRequest { required string query = 1； optional int32 page_number = 2; optional int32 result_per_page = 3; }

上述定义了一个message，包括三个字段 query page_number result_per_page,第一个是string类型
后两个是int32类型，也可以定义其他类型如enum，甚至嵌套定义message。值得注意的是类型前边的说明
required optional以及这里没有出现的repeated， required表明的是这个message在使用的时候必须初始化给定
内容的字段， optional表示的是这个字段可以出现0或者1次，repeated表示的是字段内容可以出现多次，会对应到
相应的数组，字段是有顺序的。
等号后面的数字是message在编码的时候的顺序，越小相对使用的位数相对少（1-15 只用1byte就可以，16-2047使用
2byte），因此在编码的时候应该尽量把1-15留给相对使用频繁的字段。最大的编号是2^29-1,当然一般你用不了这么多。
除此之外，19000-19999你也不能用，他们被protobuf留用了。
上述编码会遗留一个问题，就是我要尽量把序号考前的留给常用的字段，如果目前我们还没有这样的字段，可未来指不定就
有这样的字段呢？或者我们现在使用的字段在未来不使用了，可以删去了，这样在使用proto的时候，如果编号串了的话就会带
来不小的问题。因此protobuf提供了reserve的方法来缓解这种情况。如下面所示。
message Foo { reserved 2, 15, 9 to 11; reserved "foo", "bar"; }
protoc –proto_path=src –cpp_out=build/gen src/foo.proto
就可以生产两个文件foo.pb.cc foo.pb.h, 你所定义的类都在里面。

如果你在文件头写上package foo.bar;你所定义的所有类都会出现在namespace foo::bar里面。

上述message 会生成SearchRequest类，并且生成
Foo()； ～Foo()； Foo(const Foo& other)； Foo& operator=(const Foo& other)； void Swap(Foo* other)； const UnknownFieldSet& unknown_fields() const； UnknownFieldSet* mutable_unknown_fields()；
当出现嵌套的message时候如message Foo { message Bar { } }， protobuf编译器会生成两个类Foo 和 Foo_Bar，并且
在类Foo中typedef Foo_Bar Bar;并且在生成的文件中你可以Foo 和 Foo：：Bar来访问两个类。

  1. 针对optional int32 foo_bar = 5;编译器也会生成static const int kFooBarFieldNumber = 5;
  2. 针对optional 和 required字段，类型为int32 float，例如optional int32 foo = 1; required int32 foo = 1;
    编译器会生成
    bool has_foo() const; // 如果这个字段设定了，返回true
    int32 foo() const; //返回foo当前的值
    void set_foo(int32 value)； //设定foo的值
    void clear_foo()；//清空foo字段，其值返回default值，has_foo返回false。

  3. 针对optional和required字段，类型为string bytes时候
    optional string foo = 1;
    required string foo = 1;
    optional bytes foo = 1;
    required bytes foo = 1;
    会生成方法

    bool has_foo() const；
const string& foo() const；
void set_foo(const string& value)；
void set_foo(const char* value)；
void set_foo(const char* value, int size)；
string* mutable_foo()；
void clear_foo()；
void set_allocated_foo(string* value);
string* release_foo();

  4. 而针对repeated字段 如repeated int32 foo = 1；

    int foo_size() const；
int32 foo(int index)；
void set_foo(int index, int32 value)；
void add_foo(int32 value)
void clear_foo()

    当然string对象会比int对象多一些方法可以类比2和3。

  5. parse 和　serialize
    　　　 c++下
    bool SerializeToString(string* output) const
    bool ParseFromString(const string& data);

    bool SerializeToOstream(ostream* output)

    bool ParseFromIstream(istream* input);

    python下
    SerializeToString()
    　　ParseFromString(data)

生成了pb.h 和pb.cc后就可以在程序中使用了
如main.cpp 内容如下
`

#include<iostream>
#include<string>
#include<fstream>
#include<foo.pb.h>

int main()
{
foo::bar::SearchRequest sr;
sr.set_query("query_1");
sr.set_page_number(10086);

std::ofstream file;
file.open("query.file");

if(file.is_open())
{
    sr.SerializeToOstream(file);
}
else
{
    std::cout << "failed to open the file" << std::endl;
}
return 0;    
}
`

g++ main.cpp foo.pb.cc -I -lpthread -lprotobuf

sq的内容即可serial到query.file中，这个文件是二进制文件。

from　test_pb2 import * sr = SearchRequest() with open('query.file') as f: sr.ParseFromString(f.read()) print sr.page_number print sr.query
